# tele v1.0.2

This patch release makes build provenance truthful across every supported
installation path. Binaries installed through the Go module proxy now report
their module version instead of the development placeholder; checkout builds
fall back to embedded VCS revision metadata, while release archives retain their
exact workflow-stamped commit.

The README now recommends a pinned Go install before `@latest` and includes
copy-paste checksum and GitHub attestation verification for release archives.
The CLI contract, `tele/v1` machine schema, and platform support are unchanged.
